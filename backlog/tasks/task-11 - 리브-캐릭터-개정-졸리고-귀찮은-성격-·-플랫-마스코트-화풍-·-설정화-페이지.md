## Implementation Notes

<!-- SECTION:NOTES:BEGIN -->
사용자와 로컬 미리보기(tailscale)로 4라운드 반복해 확정했다.

**성격**: 나른함을 무기력이 아니라 효율의 동기로 잡았다("게으른 사람이 결국 제일 빠른 길을 찾는다"). 초안은 문단마다 귀찮음을 어필해 사용자 피드백("한 가지 캐릭터성을 계속 강조하면 쉽게 질린다")을 받고 4곳으로 줄였다. 이 빈도 규칙을 doc-3 §3-1·AGENTS.md 에 명문화했다.

**화풍**: 3안(플랫 마스코트 / 수채·색연필 / 90s 레트로 셀화)을 복장과 함께 비교 페이지로 띄워 A안(플랫 마스코트) + B안 복장(후디)이 선택됐다. 사람 비율 상반신 컷은 빼고 치비로 통일하되 설정화 2컷을 추가하기로 했다.

**생성 함정 3가지** (doc-3 §4-2·§4-3 에 반영):
- `codex exec -i ref.png "프롬프트"` 는 `-i` 가 가변 인자라 프롬프트를 삼킨다. `--` 로 끊어야 한다. 이걸 몰라 첫 4건이 통째로 빈 응답이 됐다.
- "2등신"을 강하게 누르면 다리가 뭉툭하게 짧아진다. 비율은 수치보다 참고 이미지로 잡아야 안정적이다.
- 의상만 바꿀 때 스펙만 나열하면 표정·비율이 같이 흔들린다. "인물·화풍·비율·표정·포즈는 그대로 두고 의상만 갈아입힌 컷"으로 지시해야 한다.

**임베드 사고**: placeholder 를 `IMG_OL`/`IMG_OLC` 로 지어 순차 치환했더니 앞엣것이 뒤엣것의 접두사라 3번째 컷의 data URI 끝에 `C` 가 붙어 깨졌다. 잔여 placeholder 수만 세는 검증으로는 못 잡았다. 이후 `@@SLOT0@@` 형태로 바꾸고 임베드 후 각 URI 를 디코드해 WebP 헤더까지 확인하는 절차를 AGENTS.md 에 추가했다.

**보존**: 구 시트 8컷·구 확정본·구 아바타·구 소개 페이지를 `backlog/assets/liv/legacy/` 로 `git mv` 했다(사용자 선택: repo 안에만 보존, 공개 URL 없음).
