## Implementation Notes

<!-- SECTION:NOTES:BEGIN -->
사용자와 로컬 미리보기(tailscale)로 4라운드 반복해 확정했다.

**성격**: 나른함을 무기력이 아니라 효율의 동기로 잡았다("게으른 사람이 결국 제일 빠른 길을 찾는다"). 초안은 문단마다 귀찮음을 어필해 사용자 피드백("한 가지 캐릭터성을 계속 강조하면 쉽게 질린다")을 받고 4곳으로 줄였다. 이 빈도 규칙을 doc-3 §3-1·AGENTS.md 에 명문화했다.

**화풍**: 3안(플랫 마스코트 / 수채·색연필 / 90s 레트로 셀화)을 복장과 함께 비교 페이지로 띄워 A안(플랫 마스코트) + B안 복장(후디)이 선택됐다. 사람 비율 상반신 컷은 빼고 치비로 통일하되 설정화 2컷을 추가하기로 했다.

**생성 함정 3가지** (doc-3 §4-2·§4-3 에 반영):
- `codex exec -i ref.png "프롬프트"` 는 `-i` 가 가변 인자라 프롬프트를 삼킨다. `--` 로 끊어야 한다. 이걸 몰라 첫 4건이 통째로 빈 응답이 됐다.
- "2등신"을 강하게 누르면 다리가 뭉툭하게 짧아진다. 비율은 수치보다 참고 이미지로 잡아야 안정적이다.
- 의상만 바꿀 때 스펙만 나열하면 표정·비율이 같이 흔들린다. "인물·화풍·비율·표정·포즈는 그대로 두고 의상만 갈아입힌 컷"으로 지시해야 한다.

**임베드 사고**: placeholder 를 `IMG_OL`/`IMG_OLC` 로 지어 순차 치환했더니 앞엣것이 뒤엣것의 접두사라 3번째 컷의 data URI 끝에 `C` 가 붙어 깨졌다. 잔여 placeholder 수만 세는 검증으로는 못 잡았다. 이후 `@@SLOT0@@` 형태로 바꾸고 임베드 후 각 URI 를 디코드해 WebP 헤더까지 확인하는 절차를 AGENTS.md 에 추가했다.

**보존**: 구 시트 8컷·구 확정본·구 아바타·구 소개 페이지를 `backlog/assets/liv/legacy/` 로 `git mv` 했다(사용자 선택: repo 안에만 보존). 다만 Pages 가 main 루트를 통째로 서빙하므로 `backlog/` 아래 파일도 직접 URL 로는 열린다 — 사용자에게 정정 보고했고 "그대로 둔다"로 결정했다(사이트 내비게이션에는 노출되지 않고, public repo 라 어차피 GitHub 에서 보이는 파일이다).

**후속 2(사용자 요청)**: 머그가 5컷에 반복돼 소품이 지루해졌다는 지적으로, 커피는 OL 룩 카디건 컷 한 장에만 남기고 나머지 4컷(후디·OL·사람 비율 2컷)을 머그 없이 다시 뽑았다. 태블릿 컷은 팔을 벌려 들어 실루엣이 옆으로 넓어지는 바람에 혼자 세로가 짧았는데, "태블릿을 몸 앞 가까이 붙여 든다"로 고쳐 다른 전신 컷과 같은 비율(세로/가로 2.44)이 됐다. 여기에 더해 SD 6컷을 후처리로 정규화했다 — 인물 높이를 1240px 로 통일하고 660×1300 캔버스에 발 높이를 맞춰 하단 정렬해서, 그리드에서 인물 크기와 이미지 높이가 모두 일치한다.

**금지 목록 폐기(사용자 지시)**: 설정화 페이지의 스펙 표를 없애고 doc-3 의 금지 목록도 전부 지웠다. 사용자 지적은 "일시적으로 바꿔달라고 한 걸 설정으로 편입시키면 안 된다"였다 — "다크서클·축 늘어진 자세·게슴츠레한 눈 금지"는 이번 컷이 과하게 나왔을 때의 일회성 조정이었는데 내가 영구 제약으로 박아 넣었고, 교복·안경 같은 항목까지 묶어 캐릭터가 쓸 수 있는 표현을 좁혔다. 이제 doc-3 은 기본값만 적고 "금지 목록은 두지 않는다"를 명시한다. 프롬프트에는 생성 아티팩트 방지(이미지 안 글자, 3D 질감)만 남겼다.

**후속(사용자 요청)**: 사람 비율 2컷을 후디에서 OL 룩으로 다시 뽑았다. SD 기본은 후디, 사람 비율은 OL 룩으로 갈린다. 이때 상반신 컷이 "상반신(허리 위)" 지시에도 전신으로 나와, 크롭도 부정형("허리 아래는 프레임 밖, 다리·신발 보이면 실패")으로 눌러야 한다는 것을 doc-3 §4-3 에 추가했다.
