# start-topic

화제 하나를 **주제 기록 → 대화로 논지 발전 → 보조 에이전트 검증 → 초안**까지 이끄는 글쓰기 워크플로. 컨텍스트가 리셋된 직후에도 이 스킬만으로 맥락을 복원해 시작할 수 있다.

이 스킬의 출구는 초안까지다. **게시는 하지 않는다** — 완성된 초안은 `/publish-til`(정본 절차는 `AGENTS.md` 퍼블리시 런북)로 넘긴다.

## 0. 맥락 복원 (항상 먼저)

- `AGENTS.md` 의 **"글쓰기 워크플로"** 섹션과 **퍼블리시 런북**을 읽는다. 스타일 규칙·화자 규칙·디렉터리 규칙의 정본이다.
- 진행 상황은 backlog 에서 본다: `backlog task list --plain`(`--plain` 필수, board·browser 금지). 쓰던 글이 있으면 관련 태스크가 남아 있다.
- `topics/`·`drafts/` 와 각 `_TEMPLATE.md` 존재를 확인하고 없으면 만든다. **두 디렉터리는 `.gitignore` 로 비커밋**이라 머신 간 동기화가 안 된다 — 다른 머신에서 쓰던 글이 여기 없을 수 있고, 그건 정상이다.
- 이 스킬은 리셋 직후 호출될 수 있다. 위를 읽기 전엔 규칙을 안다고 가정하지 말 것.

## 1. 화제 확정

- 인자로 화제가 주어졌으면 출발점으로 쓰고, 없으면 한 줄로 묻는다.
- 화제를 "한 줄 화제"(글이 말하려는 바, 한 문장)로 다듬어 사용자에게 확인받는다.
- `slug`(영문 kebab-case)와 `category`(예: 게임 | 기술 | 정치 | 에세이)를 정한다. slug 는 나중에 URL(`<YYYY>/<slug>/`)에 그대로 박히므로 여기서 신중히 고른다.

## 2. topics/ 기록

- `topics/_TEMPLATE.md` 를 복사해 `topics/<slug>.md` 생성.
- frontmatter(title/slug/status: topic/date/category/tags)와 "한 줄 화제", 동기, 떠오르는 논점을 채운다.
- 여러 세션에 걸칠 글이면 backlog 태스크도 하나 만든다(`backlog task create "글: <제목>" -d "..." --ac "..."`). 초안 파일이 비커밋이라 **backlog 만이 진행 상황의 추적 가능한 흔적**이다.

## 3. 대화로 논지 발전 (핵심 · 인터랙티브)

- **능동적으로 제안하고 반문한다.** 동의하거나 받아 적기만 하지 말 것:
  - 가능한 논지 2-3개와 각 장단점을 먼저 제시한다.
  - 핵심 주장 한 문장을 사용자와 함께 확정한다.
  - 글의 구조(개요) 초안을 제안한다.
- 사용자와 주고받으며 논지·구조를 다듬는다. 합의될 때까지 반복.
- 이 단계는 메인 대화에서 진행한다. 서두르지 말고 충분히 캐묻는다.

## 4. 보조 에이전트

기본은 Claude 서브에이전트다. 서브에이전트 이름은 이 저장소 규칙대로 `liv-<model>`(표시는 `🔖 리브(Opus)`), 동시에 여럿이면 `liv-2-<model>`.

외부 CLI(codex)를 쓸 수 있으면 **반론·검증은 초안을 쓴 모델과 다른 모델에게 맡기는 쪽이 낫다**(자기 논지를 자기가 봐주는 편향 방지). 외부 CLI 가 없어도 전 역할이 Claude 서브에이전트만으로 돌아가야 한다.

| 역할 | 1순위 | 폴백 | 비고 |
|---|---|---|---|
| 1. 반론·비판 (악마의 대변인) | codex | 서브에이전트 | **필수.** 검색 불필요, 논리 공격에 집중 |
| 2. 사실·근거 검증 | codex `--search` | 서브에이전트(웹 검색) | 사실 주장이 있는 글이면 사실상 필수 |
| 3. 다각도 관점 확장 | 서브에이전트 | — | 필요 시 |
| 4. 문장·표현 다듬기 | 서브에이전트 (고정) | — | 초안 작성 후. 문체 모사 맥락이 필요해 외부 CLI 부적합 |

codex 비대화형 호출(파일 수정 없이 텍스트 응답만 받는다):

```bash
cat role-prompt.md body.md | codex exec - --output-last-message out.md   # 검증 역할이면 --search 추가
```

- til 저장소가 codex trusted 가 아니면 `--skip-git-repo-check` 를 붙인다.
- 입력은 frontmatter·대화 로그·작업 메모를 뺀 **논지 요약 또는 본문만** 추려 scratchpad 임시 파일로 만든다.

### 역할 프롬프트

공통 머리말(모든 역할 프롬프트 맨 앞):

> 너는 글쓰기 프로젝트의 보조 리뷰어다. 아래 지시에 대한 텍스트 응답만 하고, 파일 생성·수정·명령 실행은 하지 마라. 한국어로 답하라. 아부하지 말고, 지적할 게 없으면 없다고 말하라.

1. **반론·비판**:
   > 역할: 악마의 대변인. 아래 글(논지)의 허점을 공격하라. (1) 논리적 비약·숨은 전제 (2) 강력한 반례 (3) 반대 진영에서 나올 가장 센 반론 을 각각 찾고, 심각도(상/중/하)를 매겨라. 글을 고쳐 쓰지 말고 공격만 하라. 각 항목은 3문장 이내.

2. **사실·근거 검증**:
   > 역할: 팩트체커. 아래 글에서 사실 주장(수치·연도·인명·사건·인과 주장)을 모두 추려 목록화하고, 각각 검증 결과를 [확인됨/틀림/확인불가] 로 표시하라. 틀림·확인불가는 근거(출처 URL 또는 판단 이유)를 한 줄로 달아라. 의견·취향 문장은 건드리지 마라.

3. **다각도 관점 확장**:
   > 역할: 관점 확장가. 아래 글이 다루지 않은 시각을 서로 다른 분야에서 3개 제시하라(예: 경제학/법/역사/기술/당사자 경험). 각 관점은 "이 글에 넣으면 논지가 어떻게 달라지는가" 한 줄을 포함해 3문장 이내로. 글의 결론에 동조할 필요 없다.

4. **문장·표현 다듬기** (Claude 서브에이전트 전용):
   > 역할: 문장 교열자. `~/work/kil9log/archived/STYLE_GUIDE.md` 와 til `AGENTS.md` 의 글쓰기 스타일 규칙(이탤릭 금지, 볼드 최소, 엠대시 금지, 자문자답 편당 1회, 짧게·아이디어 위주)을 기준으로 아래 글을 교열하라. 규칙 위반·어색한 문장만 "원문 → 수정안 (이유)" 형식으로 목록화하고, 전체를 다시 쓰지 마라.

### 결과 취합 (drafts 대화 로그)

외부 CLI 든 서브에이전트든 결과는 같은 형식으로 `drafts/<slug>.md` 의 `## 대화 로그 / 정리` 아래에 수렴시킨다. **원문 응답은 로그에 붙여넣지 않고**(로그 비대화 방지) scratchpad 에만 두며, 로그에는 판정을 끝낸 정리본만 남긴다.

1. **판정**: 항목별 채택 여부를 정한다. 반론이면 흡수 방식(정면 방어/완화/구조 반영)까지, 검증이면 교정 여부까지. 기계적으로 다 받지 말고, 기각에도 이유를 단다. 받아온 반론은 사용자에게 정리해 보여주고 함께 판단한다 — 무비판 수용도 무시도 하지 않는다.
2. **기록**: 섹션 제목은 `### <역할> (<에이전트>, <날짜>)`, 첫 줄에 요약 통계("9건(상 5, 중 4) 중 흡수 5, 완화 2, 기각 2"), 항목마다 판정 태그(`[상·흡수]` `[중·완화]` `[기각]` `[교정]` `[보류]`)를 앞에 단다.
3. **반영**: 채택분을 논지·구조·초안에 반영한다. 큰 방향 전환이면 로그에 한 줄 남긴다("논지 축 이동: A → B").

```markdown
### 반론·비판 (codex, 2026-07-19)

3건(상 2, 중 1) 중 흡수 2, 기각 1.

- [상·흡수] 숨은 전제 공격: 교육=기술 전수라는 전제가 무너지면 논지 전체가 무너진다. → 교육 목적을 시각적 문해력으로 재정의하는 반론을 본문에 선제 배치.
- [기각] 도구 접근성 격차 지적: 이 글의 범위 밖. 별도 글감으로 backlog draft.
```

## 5. 초안 작성

- `drafts/_TEMPLATE.md` 를 복사해 `drafts/<slug>.md` 생성.
- 논지(한 문장) / 대화 로그 / 구조(개요) / 본문 초안을 채운다. frontmatter 의 `source_topic` 에 topics 파일을 연결하고 `status: drafting`.
- **화자는 사용자 본인**이다(decision-6). 리브 문체로 쓰지 않고, 리브 코멘트·삽화는 기본적으로 얹지 않는다.
- 문체는 `~/work/kil9log/archived/STYLE_GUIDE.md`(공통), 게임 리뷰면 같은 폴더의 `GAME_REVIEW_STYLE_GUIDE.md` 를 추가로 참고한다. 근거 해부는 `STYLE_ANALYSIS.md`, 원자료 코퍼스는 `archived/posts/`. **이 파일들은 개인 글이라 til 에 복사·인용하지 않는다** — 읽고 반영만 한다. 없는 머신이면 til `AGENTS.md` 의 스타일 규칙 1-5 만으로 쓰고, 문체 모사가 약해진다는 점을 사용자에게 알린다.

## 6. 마무리

- **커밋하지 않는다.** `topics/`·`drafts/` 는 gitignore 대상이라 커밋할 것이 없다. backlog 태스크를 만들었으면 그 상태만 갱신하고 커밋한다.
- 사용자에게 다음 선택지를 제시한다: 초안 더 다듬기 / 문장 다듬기 에이전트 투입 / `/publish-til` 로 게시.
- 게시로 넘어가면 `AGENTS.md` 퍼블리시 런북이 정본이다(HTML 셸·favicon·OG·beacon·갤러리 카드·README 표·주제 칩).
